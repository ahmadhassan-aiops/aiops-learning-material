from selenium import webdriver
from selenium.webdriver.support.wait import WebDriverWait
from webdriver_manager.firefox import GeckoDriverManager
from webdriver_manager.chrome import ChromeDriverManager
from selenium.webdriver.support import expected_conditions

from selenium.webdriver.common.by import By
import time

# Firefox
driver = webdriver.Firefox(executable_path=GeckoDriverManager().install())

# Google
# driver = webdriver.Chrome(executable_path=ChromeDriverManager().install())


# Declaration
wait = WebDriverWait(driver, 20)  # Declaration

driver.get('https://www.google.com/intl/en-GB/gmail/about/')
driver.maximize_window()

driver.find_element(By.PARTIAL_LINK_TEXT, 'Sign in').click()
driver.find_element(By.ID, 'identifierId').send_keys("testautomation52786@gmail.com")

# Next button for verifying your email address
driver.find_element(By.XPATH,
                    '/html/body/div[1]/div[1]/div[2]/div/c-wiz/div/div[2]/div/div[2]/div/div[1]/div/div/button/span').click()

action = wait.until(expected_conditions.element_to_be_clickable((By.NAME, 'Passwd')))
action.click()

# Password typing

print("Login success")

Implicit wait :-

1. It is global wait and applied to all emenets i n the webpage 
2. Specific amount of time to wait the load of DOM page.
3. Its does not required any condition.
4. Its very simple because we need to declare at starting of the code.
5. its affect the execution speed since each steps waits for elements to be ready



Explicit wait :-
1. its not global wait and applied to perticuler scenario.
2. the driver is asked to wait till a certains condition is match.
3. its required condition then we will applied 
4. its lit bit complex because we need declare of time to wait and we need to applied that condition of perticuler elements.
5. its does not affect the execution speed since its applicable to perticuler elements of the page.
